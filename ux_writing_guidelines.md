# UX Writing Guidelines
*Part of the Chiti Technologies V3 Ecosystem*

## 1. Core Principles
The interface copy is the voice of the product. If the design is the skeleton, the copy is the personality. We adhere to three rules:
1. **Clear over Clever:** Users want to accomplish tasks, not solve riddles. Be direct.
2. **Concise but Human:** Remove unnecessary words, but do not sound like a robot.
3. **Action-Oriented:** Start headers and buttons with strong verbs.

---

## 2. Button and Action Labeling
Never use passive or generic words like "Submit", "Click Here", or "OK". Use specific verbs that describe exactly what will happen.
- **Bad:** `Submit` | **Good:** `Create Account`
- **Bad:** `Yes` | **Good:** `Delete Workspace`
- **Bad:** `OK` | **Good:** `Save Configuration`

---

## 3. Voice & Tone by Vertical

### Enterprise / SaaS (e.g., NetQ Command, BatchFlow)
- **Voice:** Confident, precise, and authoritative.
- **Tone:** Neutral and efficient. Do not use exclamation points or slang.
- **Example:** "Data export initiated. You will receive an email when processing is complete."

### Commerce / D2C (e.g., BazaarSetu, Bighi Brothers)
- **Voice:** Aspirational, engaging, and trustworthy.
- **Tone:** Enthusiastic but secure. Use positive reinforcement during checkouts.
- **Example:** "Your premium collection is reserved. Let's get this shipped to you!"

---

## 4. Writing Error Messages
Error messages must never blame the user. They must explain what happened and provide a clear path forward.
- **Formula:** `[What went wrong] + [Why it happened] + [How to fix it]`
- **Bad:** "Invalid Input: File too large."
- **Good:** "We couldn't upload this image. It exceeds the 5MB limit. Please compress the file and try again."
