# Iconography & Asset Pipeline
*Part of the Chiti Technologies V3 Ecosystem*

## 1. Icon Grid & Geometry
All icons within the Chiti ecosystem must conform to a strict geometric foundation to ensure visual harmony alongside our typography.
- **Base Grid:** 24x24 pixels.
- **Live Area:** 20x20 pixels (2px inner padding prevents optical crowding).
- **Stroke Weight:** Exactly `1.5px` (This strikes the perfect balance between the bold `Outfit` headers and thin `Inter` body text).
- **Corner Radius:** `2px` for external container corners; `0px` (sharp) for internal structural overlaps.

## 2. Asset Export Standard
Icons are never directly exported as `.PNG` or `.JPG`. All vector assets must be exported from Figma as raw `.SVG` files.

## 3. The React Conversion Pipeline (SVGR)
Developers are forbidden from embedding raw SVG code inline into JSX. Instead, the `@svgr/cli` pipeline automatically converts SVG assets into strictly typed React components.

```tsx
// Bad
const IconView = () => <svg>...</svg>;

// Good (Auto-generated by pipeline)
import { EyeIcon } from '@chiti/ui/icons';

<ChitiButton icon={<EyeIcon />}>
  View Details
</ChitiButton>
```

## 4. Dynamic Coloring
Never bake `fill` or `stroke` colors into the SVG markup. Ensure all SVGs are set to `stroke="currentColor"`. This forces the icon to inherit the text color of its parent container, allowing our Dark Mode/Light Mode CSS variables to automatically control icon visibility.
