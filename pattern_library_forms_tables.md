## 📋 1. Pattern: Form Orchestration (The "Command" Layout)
Forms in Chiti V3 prioritize **Cognitive Load Reduction** and **Tactile Feedback**.

### 📐 Structural Pattern: Single-Column (Standard)
*Best for: Onboarding, Account Settings, Data Entry.*
```text
[ LABEL ]
[ INPUT (space-1 gap) ]
(space-4 Gap)
[ LABEL ]
[ INPUT (space-1 gap) ]
(space-6 Gap)
[ PRIMARY ACTION ] [ SECONDARY ACTION ]
```

### 🧩 Best Practices
- **Real-Time Validation**: Errors must trigger **Instantly** on `blur`. Use `semantic.error` for text and border.
- **Haptic Audio Ticker**: Every keypress should trigger a `10ms @ 440Hz` tick (sub-perceptual) to reinforce the "Mechanical" feel of the system.
- **Micro-Copy**: Labels use `Inter (Weight: 600)`. Placeholder text must never contain critical information.

## 📊 2. Pattern: Data Table Engineering (The "Telemetry" Grid)
Tables are the heart of `NetQ Command`. They must remain readable under massive data loads.

### 📐 Structural Pattern: Zebra Density (Industrial)
```text
/------------------ STICKY HEADER (Glassmorphic) ------------------\
|  [ NAME ]     [ STATUS ]     [ TELEMETRY ]     [ ACTIONS ]     |
|------------------------------------------------------------------|
|  Row 1 (Surface-1) | ACTIVE | [#######---] | [ Edit ] [ Del ]  |
|  Row 2 (Surface-2) | IDLE   | [##--------] | [ Edit ] [ Del ]  |
\------------------------------------------------------------------/
```

### 🧩 Interaction Specs
- **Row Hover**: 300ms Spring-Ease transition. Background shifts from `surface-1` to `surface-2`.
- **Skeleton State**: Rows must shimmer while API data hydrates. Use 8pt grid-height segments.
- **Truncation**: Critical metrics must never wrap. Use "Executive Truncation" (Ellipsis) with `Tooltip` reveal on hover.
